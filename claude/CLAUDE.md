# 전역 지침

## 언어
- 모든 답변과 설명은 한국어로 한다. 코드 식별자, 기술 용어, 커밋 메시지는 원문(영어) 유지.

## 작업 방식
- 수정 전에 관련 코드와 기존 컨벤션을 먼저 읽는다. 추측하지 말고 근거를 확인한다.
- SDK/프레임워크/API 사용이 불확실하면 공식 문서를 먼저 확인한다.
- 완료라고 말하기 전에 빌드·테스트·실행으로 실제 동작을 검증한다.
- 커밋과 푸시는 명시적으로 요청받았을 때만 수행한다.

## 코드 스타일
- 기존 코드베이스의 스타일, 네이밍, 주석 밀도를 따른다.
- 요청 범위를 벗어난 리팩터링이나 불필요한 추상화를 추가하지 않는다.

## 로컬 환경
- Java: openjdk@21 (JAVA_HOME 설정됨), 빌드는 maven
- Node: brew node@22 / Python: uv (기본 python3=3.14, 프로젝트는 .venv) / Go: GOPATH=~/Library/Go
- 컨테이너: colima + docker CLI / Kubernetes: `k`=kubectl alias 사용
